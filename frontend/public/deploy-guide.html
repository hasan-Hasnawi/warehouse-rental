1 شراء وتجهيز الـ VPS

اتبع الخطوات في لوحة تحكم هوستنغر (hPanel):

  1. سجل الدخول إلى hPanel → اختر VPS.
  2. اختر الخطة KVM 1 (أو أعلى حسب احتياجك).
  3. اختر نظام التشغيل: Ubuntu 22.04 LTS.
  4. اختر أقرب خادم جغرافي لجمهورك المستهدف (مثلاً: Europe للعراق/السعودية أو Middle East إن وجد).
  5. أكمل عملية الشراء — ستحصل على IP وبيانات SSH.
🔑 بيانات SSH: ستصلك عبر البريد: IP السيرفر، اسم المستخدم (root)، وكلمة المرور.

2 الاتصال بالسيرفر

استخدم أي عميل SSH (Terminal في Mac/Linux، أو PuTTY/Windows Terminal في Windows):

# على ويندوز استخدم PowerShell:
ssh root@XX.XXX.XX.XX

# ستُطلب منك كلمة المرور — اكتبها (لن تظهر)

بعد أول تسجيل دخول، حدّث النظام:

sudo apt update && sudo apt upgrade -y

7 إعداد Nginx (Reverse Proxy + Domain + SSL)

7.1 تكوين Nginx لاستضافة المواقع

أنشئ ملف إعدادات لموقعك:

sudo nano /etc/nginx/sites-available/warehouse-rental

ضع هذا المحتوى (استبدل your-domain.com باسم مجالك):

server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # Frontend (Next.js)
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # Backend (API)
    location /api/ {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # زيادة حجم الملفات المسموح بها للـ upload
        client_max_body_size 50M;
    }

    # WebSocket (إذا كان مستخدماً)
    location /socket.io/ {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

فعّل الموقع:

sudo ln -s /etc/nginx/sites-available/warehouse-rental /etc/nginx/sites-enabled/
sudo nginx -t          # اختبار الإعدادات
sudo systemctl reload nginx

7.2 تفعيل SSL (مجاناً مع Certbot)

sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

اتبع التعليمات — ستُحدّث إعدادات Nginx تلقائياً مع SSL.

🔒 تهانينا! موقعك الآن يعمل عبر HTTPS. سيتجدد الشهادة تلقائياً كل 90 يوماً.

8 ربط اسم المجال

  1. اذهب إلى إعدادات DNS في خدمة الدومين (أو في hPanel).
  2. أضف سجل A للقيمة @ وعنوان IP السيرفر الخاص بك.
  3. أضف سجل CNAME للقيمة www إلى @.
  4. انتظر 5–30 دقيقة لنشر التغييرات.
النوعالاسمالقيمة
A@XX.XXX.XX.XX (IP السيرفر)
CNAMEwww@

9 إعداد البريد الإلكتروني (SMTP)

لإرسال إشعارات البريد (تأكيد التسجيل، الفواتير، إلخ).

الخيار الموصى به: Resend (مجاني للبدء)

  1. سجل في resend.com.
  2. أنشئ API key.
  3. أضف المجال الخاص بك للـ verification.
  4. حدّث ملف .env في الباكند بقيم SMTP.

بدائل: SendGrid (100 إيميل/يوم مجاناً)، Mailgun، أو SMTP2GO.

10.4 نصائح نهائية

  • النسخ الاحتياطي: استخدم pg_dump لعمل نسخة احتياطية يومية لقاعدة البيانات
  • جدار الحماية: أعدد ufw لفتح المنافذ 80، 443، 22 فقط
  • التحديثات: شغّل sudo apt update && sudo apt upgrade -y أسبوعياً
  • Fail2ban: ثبّته للحماية من هجمات SSH
# إعداد UFW (جدار الحماية)
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw --force enable
sudo ufw status

# تثبيت Fail2ban
sudo apt install -y fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

+ استكشاف الأخطاء الشائعة

المشكلة: 502 Bad Gateway

السبب: Nginx لا يصل إلى التطبيق. تأكد من أن PM2 يشغل التطبيقات (pm2 list).

المشكلة: Cannot connect to database

السبب: PostgreSQL لا يعمل أو بيانات الاتصال خاطئة. تحقق:

sudo systemctl status postgresql
# تأكد من صحة DATABASE_URL في .env

المشكلة: SSL not renewing

الحل: اختبر التجديد يدوياً:

sudo certbot renew --dry-run

المشكلة: Next.js 502 بعد الـ build

السبب: ربما نسخة Node.js قديمة. تأكد من أنك تستخدم Node 22.x

node -v
# إذا أقدم من 18، أعد تثبيت Node.js
